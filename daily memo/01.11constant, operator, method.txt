상수 - 변수와 같이 데이터를 저장하는 공간. 단, 한 번 저장된 데이터를 수정/변경할 수 없다.
	원래는 변수인데, 값을 초기화 하면 상수화가 된다 
	오염되지 않을 값을 담는 변수 = 상수  
	기본적으로 변경되면 안되는 값 ( 환경 설정 값 - 서버 IP, DB 접속 정보 등 )

상수 사용 - 변수와 거의 동일하나, 선언할 때 final 이라는 예약어를 붙인다. 
# 관례상 상수명은 전부 대문자로 사용, 두 단어가 합성어일 때는 Snake case 로 작성한다 
 final int A ;   

오버플로우 : 자료형이 가질 수 있는 최대값을 넘어설 경우, carry 값을 제거하고 부호비트를 변경하여 + - 를 변경한다  

언더플로우 : 오버플로우와 반대로 최소값을 넘어설 경우 carry 값을 제거하고 부호비트를 변경하여 - + 를 변경한다 

형변환 : 정수는 실수로 자동으로 형 변환 가능 , char 도 int로 자동 형변환 가능 

연산자 
우선순위 
1.소괄호 2. 단항 연산자 3.다항 연산자(이항 > 삼항) 4. 대입 연산자  
 

메소드 : 일반적인 프로그래밍 언어에서 기능을 하는 것을 '함수' 라 부른다. 자바에서는 '메소드'라 부름 
	특정 작업을 하기 위한 명령문의 집합 
	모든 메소드는 class 안에 존재 
	기능 - 메소드, 도화지 - class 

컴퓨터는 프로그램이 처음 실행되었을때 class 이름 이랑 static이 붙어있는 method들만 먼저 인식을 한다. 
-> static 을 붙이지 않는 경우 

API 
OS 위에 돌아가는 어플리케이션을 개발할 때 필요한 인터페이스 (매개체) 

특정 기능이 재사용 되겠다 싶으면 해당 부분을 메소드로 만들어서 재사용 할 수 있게끔 한다 -> 모듈화 라고도 함
