웹 브라우저의 한 페이지에 들어갈 테이블은 관계를 맺을 수 있도록 ( 조인 가능하도록) 


session
서버가 만료시간동안 뭔가 정보를 들고 있는 것 
쿠키의 취약점 : 클라이언트 쪽에서 정보를 관리함 -> 정보 유출의 가능성이 있음 (보안성 취약) 
		     사용하는 PC가 변경되었을 때 상태값을 저장하지 못함 (동기화되어 상태를 유지하기가 힘듬)

장점 : 클라이언트 입장에서는 부담이 적음 
단점 : 서버의 부하가 커짐 

세션사용이유 : 서버가 클라이언트를 제어할 수 있기 때문 
Jsessionid : 서버에서 관리하는 세션의 사용자 라는 것을 확인하기 위한 키 


filter
요청과 응답 사이에 필터를 적용해서 사용자 로그를 기록하거나 


평문 (plain text) 를 다이제스트해서 암호화 


spring 

빈은 이름으로 관리하는데, 하나의 이름에는 하나의 객체만 들어간다 ( 싱글톤 패턴으로 관리 ). 설정 변경으로 싱글톤이 아니라 다른 패턴도 설정은 가능하지만 그럴 경우 컨테이너의 부담이 커진다.
